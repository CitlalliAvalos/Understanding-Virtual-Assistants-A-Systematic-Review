# Understanding-Virtual-Assistants-A-Systematic-Review
Supplementary dataset of the article "Understanding Virtual Assistants: A Systematic Review". This repository contains the included studies, extracted text fragments, source references, analytical interpretations, and the coding of reported limitations and challenges.
The dataset is based on 34 review studies published between 2015 and 2025.
## Dataset Structure

The dataset is organized into four sheets within a single Excel file:

### 1. Extracted Evidence

This sheet contains the primary data extracted from the included studies. It includes:

- `ID`: Identifier of the study  
- `Area`: Application domain  
- `Subarea`: Specific subdomain  
- `Reference`: Bibliographic reference  
- `Limitations or challenges description`: Description of the identified issue  
- `Extracted text fragments related to limitations or challenges`: Verbatim excerpts from the source articles  
- `Location of the text within the source article`: Section or position in the original document  
- `Cross Reference`  

---

### 2. Challenges Coding

This sheet contains the coding of challenges identified in the literature. It includes:

- `Article ID`: Identifier of the source study  
- `Area`: Application domain  
- `Year of Publication`: Publication year  
- `Challenge ID`: Unique identifier of each challenge  
- `Problem to be solved`: Description of the challenge  
- `Category(1)–Category(5)`: Assigned categories (primary and secondary)  
- `Unified Concept`: Normalized concept representing the challenge  

---

### 3. Limitations Coding

This sheet contains the coding of limitations reported in the literature. It includes:

- `Article ID`: Identifier of the source study  
- `Area`: Application domain  
- `Year of publication`: Publication year  
- `ID limitation`: Unique identifier of each limitation  
- `Summary limitation`: Description of the limitation  
- `Category(1)–Category(5)`: Assigned categories (primary and secondary)  
- `Unified Concept`: Normalized concept representing the limitation  

---

### 4. Temporal Analysis

This sheet contains data used for analyzing trends across technological stages. It includes:

- `ID`: Identifier of the study  
- `Reference`: Bibliographic reference  
- `Year`: Publication year  
- `Period covered`: Time span analyzed in the study  
- `Approach`: Technological stage classification  
- `Justification`: Rationale for the assigned classification  

This sheet also includes the inclusion and exclusion criteria for assigning studies to the categories **Pre-LLM**,
**Transitional AI-based**, and **LLM-influenced**

---

## Purpose and Use

This dataset is provided to:

- support transparency and reproducibility of the review process,  
- enable validation of the analytical procedures,  
- facilitate further research on virtual assistants,  
- allow reuse of categorized limitations and challenges.
